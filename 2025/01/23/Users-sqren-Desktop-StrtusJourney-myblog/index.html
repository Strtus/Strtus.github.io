前言

Hexo 是一款基于 Node.js 的静态博客框架，能够将网站内容预先渲染为 HTML 文件，从而避免了服务器实时渲染的需求。这与传统的动态博客不同，后者依赖于数据库和服务器端脚本来动态生成内容。因此，静态博客具备更快的加载速度、较低的服务器资源消耗和更高的安全性

由于内容以静态文件形式存储和分发，静态博客可以通过如 GitHub Pages 等托管服务实现免费部署，降低搭建成本

此外，Hexo 对 Markdown 和 LaTeX 的支持非常友好，渲染速度快，尤其适合构建技术型或知识分享类博客

安装HEXO

1.在本地创建安装目录，比如这是我的目录：

通过cd命令切换到该目录：

cd /Users/sqren/Desktop/StrtusJourney/myblog

通过npm来安装hexo：

npm install -g hexo-cli

如果你和我得到了类似的output：

这是由于当前用户没有权限在 /usr/local/lib/node_modules 目录下安装全局的npm包，如果不希望在系统目录下安装 npm 包，可以将 npm 的全局安装路径修改为你自己用户目录下的某个位置

首先，创建一个目录来存放全局安装的 npm 包：

mkdir ~/.npm-global

然后，告诉 npm 使用这个新目录来安装全局包：

npm config set prefix '~/.npm-global'

接着，编辑 .bash_profile 或 .zshrc 文件，加入以下内容：

export PATH=~/.npm-global/bin:$PATH

重新加载配置文件：

source ~/.bash_profile

之后再次运行安装命令

npm install hexo-cli -g

如图：

在我们选定的安装目录下可以看到Hexo CLI：

BTW，如果你和我一样把HEXO框架安装在了iCloud/OneDrive的同步文件夹下，请记得勾选“保留下载/使用保留在此设备上”

安装完成后，初始化我们的博客，输入：

hexo init blog

不出意外的话这个时候又要出意外了

看报错信息，提示需要在 Hexo 项目的根目录下运行 npm install 来安装项目所需的依赖包

之前只是安装了 Hexo CLI，现在需要安装 Hexo 的依赖项

那么，按照它的提示👇

使用npm install命令根据 package.json 文件中的配置安装所有必需的依赖包

呐，屁用没有

根据报错信息，npm 缓存目录（/Users/sqren/.npm）中包含了由 root 用户拥有的文件，这是因为之前我使用 sudo 安装过 npm 包

现在我们需要将 /Users/sqren/.npm 目录的所有权更改为当前用户：

sudo chown -R $(whoami):$(whoami) /Users/sqren/.npm

如果给了这个报错

说明 $(whoami) 命令返回的用户名与系统中实际存在的用户组不匹配

检查当前用户的名称和对应的用户组：

id

会看到如下output：

其中：

  • uid=501(sqren) 表示当前用户是 sqren。
  • gid=20(staff) 表示当前用户的用户组是 staff。

于是修正 chown 命令：

把 .npm 目录及其所有内容的所有权更改为 sqren 用户和 staff 组。

sudo chown -R sqren:staff /Users/sqren/.npm

清空 npm 缓存以防出现奇奇怪怪的问题

npm cache clean --force

接下来就可以愉快的npm install了

安装完成后运行如下命令初始化HEXO：

HEXO init

随后启动HEXO本地服务

HEXO serve

如图

浏览器访问 http://localhost:4000

能进来这个页面就说明安装成功了

撰写新文章

cd到HEXO目录

使用new命令来新建一个带有front-matter的.md文件

HEXO new 'title'

随后你可以在posts目录下打开此文档

撰写完文章后使用如下命令清除缓存：

hexo clean

生成html文件

hexo generate

git

hexo deploy

启动本地服务：

hexo serve

即可在浏览器中访问

在仓库中获取SSH链接

在站点配置文件_config.yml中修改deplog的键值：

安装Git部署插件

npm install hexo-deployer-git --save

配置SSH-Key

ssh-keygen -t rsa -C <your mail>

如图👇：

添加生成的SSH key到ssh-agent：

ssh-add ~/.ssh/id_rsa

复制key，然后回到Github，进入路径user/Settings/Access/SSH and GPG keys

Add new SSH key

测试添加ssh是否成功

ssh -T git@github.com

如果看到Hi后面是你的用户名，就说明成功了
